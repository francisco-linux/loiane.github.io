Olá pessoal,

Hoje vou mostrar como configurar a variável de ambiente ANT_HOME.

O Apache Ant é uma biblioteca Java utilizada para automatizar builds. Muito utilizada em projetos Java e também utilizada para fazer build de projetos Android quando usamos o Phonegap (ou Cordova).

O requisito para instalar o Ant e que para que funcione também é ter o Java JDK instalado (não pode ser o Java Runtime (JRE), pois o ant utiliza o comando javac (compilar projeto java). Então, caso não tenho o Java JDK instalado, pode conferir um tutorial aqui ou vídeo-aula aqui.

Nesse tutorial vou fazer a configuração para o SO Windows 8 (a configuração é a mesma pra outras versões do Windows).

1 - Download

Para fazer o download, basta acessar o site do Apache Ant: http://ant.apache.org/. O link direto para download é esse aqui: http://ant.apache.org/bindownload.cgi

Existem 3 opções de download. Pode fazer o download de qualquer uma (zip, tar.gz, tar.bz2):

2 - Instalação

Como é um zip, basta extrair para qualquer lugar de preferência do seu computador. É importante guardar o caminho de onde foi instalado para a configuração da variável de ambiente a seguir.

3 - Criando a variável de ambiente ANT_HOME

No windows, clique com o botão direito em Meu Computador, e escolha a opção Propriedades:

Clique em configurações avançadas:

Variáveis de ambiente:

Na parte de baixo das variáveis de ambiente (não pode ser na parte de cima), clique em Novo (precisa de acesso administrador pra isso):

Entre com ANT_HOME e o caminho de onde foi extraído o Ant no seu computador:

4 - Adicionando o ANT_HOME no Path:

Com a variável ANT_HOME criada, vamos adicionar o ANT_HOME no Path. O comando ant está disponível nesse diretório e por isso temos que adicionar ele no Path.

Na parte de baixo das variáveis de ambiente, localize o Path, selecione e clique em Editar:

No final, adicione o seguinte valor:

;%ANT_HOME%\bin

Vai ficar assim (o importe é adicionar no final do Path, o que tiver antes está ok - e depois pode adicionar outras variáveis como Android, etc):

Basta clicar em OK em tudo, fechar o terminal (CMD, caso esteja aberto) e fazer o teste digitando ant. A mensagem de que o ant está configurado corretamente precisa ser algo similar a imagem abaixo:

Só não pode ter erro de comando não reconhecido, pois significa que a configuração não está completa.

Com essa configuração você consegue fazer build de projetos java automaticamente, e caso você esteja trabalhando com phonegap/cordova/ionic, você consegue fazer o build da plataforma Android!
